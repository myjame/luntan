# 公开兴趣社区 MVP 技术任务清单

版本：v0.1

日期：2026-03-26

状态：开发执行版

关联文档：

- `docs/community-mvp-spec.md`

## 1. 文档目的

本文件用于把 MVP 需求文档拆解为可直接执行的技术开发任务。

使用方式：

- 按本文的步骤顺序推进开发
- 每一步先完成数据、接口、权限，再补页面交互
- 每一步完成后，对照“完成标志”做自检
- 非明确写入本文的增强项，不默认进入首版范围

## 2. 首版开发原则

### 2.1 总体原则

- 先做可运行闭环，再做体验增强
- 先做数据结构和权限边界，再做页面细节
- 游客可浏览，登录且审核通过的用户才可互动
- 匿名内容对普通用户匿名，但对管理员必须可追溯
- 新用户前几次发帖和评论默认进入审核队列
- 删除默认做软删除，编辑默认保留历史
- 关键后台动作默认记录操作日志
- 注册、登录、发帖、评论默认需要基础限流

### 2.2 首版范围边界

以下内容不进入首版开发范围：

- 视频发布与视频审核
- WebSocket 实时推送
- 邮件通知
- 对象存储
- App 客户端
- 复杂推荐算法
- 完整增长分析看板
- 签到系统

### 2.3 技术基线

- 前后端框架：`Next.js`
- 数据库：`PostgreSQL`
- ORM：`Prisma`
- 缓存：`Redis`
- 样式：`Tailwind CSS`
- 部署：`Docker Compose`
- 文件存储：本地磁盘

## 3. 建议开发顺序

建议按以下顺序推进：

1. 项目骨架与基础设施
2. 数据模型与通用能力
3. 认证、账号状态与权限
4. 后台基础壳与审核入口
5. 圈子系统
6. 帖子、评论、投票、附件
7. 点赞、收藏、关注、主页
8. 通知、`@用户`、私信
9. 内容审核、治理与风控
10. 运营后台
11. 搜索、热度、积分、勋章、统计
12. 发布前收尾

## 4. 首条可运行闭环

如果希望尽快进入可演示状态，建议先打通以下闭环：

1. 项目骨架
2. 用户注册、审核、登录
3. 后台用户审核
4. 圈子列表、圈子详情、圈子申请
5. 发帖、评论、投票、附件
6. 举报、敏感词、新用户内容审核
7. 后台基础内容管理

在上述闭环完成前，不优先处理搜索、积分、勋章、轻量统计等增强能力。

## 5. 分步骤任务清单

### Step 1 项目骨架与基础设施

目标：建立可运行的工程骨架，为后续模块开发提供统一底座。

任务清单：

- 初始化 `Next.js` 项目，确定 `App Router` 方案
- 集成 `Tailwind CSS`
- 规划目录结构，至少区分前台、用户区、后台、服务层、数据层
- 建立全局布局、导航、页头、页脚、空态、错误态、加载态
- 初始化 `Prisma` 与数据库连接
- 初始化 `Redis` 客户端
- 建立 `Docker Compose`，至少包含 `app`、`postgres`、`redis`
- 为 `postgres`、`redis`、上传目录配置持久化挂载卷
- 配置环境变量与 `.env.example`
- 建立本地上传目录与静态文件映射策略
- 集成基础代码规范工具，如 lint、format、类型检查
- 建立通用请求封装、错误返回格式与服务端日志输出
- 建立基础 UI 组件：按钮、输入框、下拉框、弹窗、分页、表格、上传、提示消息

建议产出：

- 项目目录结构说明
- `docker-compose.yml`
- `.env.example`
- 应用基础布局和导航壳
- 数据库连接与 Redis 连接封装

完成标志：

- 本地可一键启动应用、数据库和缓存
- 首页和后台占位页可以访问
- Prisma migration 可以正常执行
- 上传目录可以正常写入和访问

## 6. 跨步骤通用约束

以下约束在所有开发步骤中都应保持一致：

- 公开内容默认允许游客浏览
- 所有互动能力默认要求登录且账号为 `active`
- 匿名内容前台不暴露真实作者
- 管理端必须具备匿名作者追溯能力
- 重要删除操作默认软删除
- 编辑类内容默认保留修订历史
- 治理动作默认写入操作日志
- 上传内容默认做类型白名单校验
- 高风险接口默认加入限流与权限校验
