# 前端视觉与交互方向

版本：v0.1

日期：2026-03-26

## 1. 目标

本项目的前端不做模板化论坛页面，而是做成有“社交娱乐感”和“内容场域感”的社区界面。

核心体验目标：

- 首屏就能让用户感知社区气质
- 广场和发现页要有内容节奏，不只是列表堆砌
- 圈子页要像“一个可以停留的房间”，而不是普通分类页
- 后台也保持清晰层次，但不过度像传统企业后台

## 2. 建议视觉方向

- 底色采用暖色纸面感和轻颗粒感，而不是冷白模板色
- 标题使用更有性格的字形和更大的版心，拉开层次
- 通过渐变、光斑和卡片高低差构建“活跃社区”的氛围
- 避免所有模块都长成同一种卡片，保持区块节奏变化

## 3. 建议交互方向

- 页面首屏区块采用分层淡入和轻位移动效
- 卡片 hover 时给轻微上浮和阴影增强，强化可探索感
- 推荐区块与运营位采用错位布局，不做纯网格流水线
- 移动端保留主次结构，不简单压平为连续白卡片

## 4. 首版可直接落地的动效

- 首屏主视觉淡入
- 渐变光斑缓慢漂移
- 卡片 hover 上浮
- 页面区块分段进入
- loading 态统一视觉语言

## 5. 实施建议

- 先建立全局设计变量：颜色、圆角、阴影、间距、字体
- 先做首页、广场、圈子页三种页面模版
- 后续各业务页面复用统一视觉基线，避免越做越散
- 在功能没完全落地前，先把页面气质和信息结构做稳
